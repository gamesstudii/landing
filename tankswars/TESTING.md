# Tanks Wars: документация для тестирования

Документ предназначен для двух ролей:

- тестеры: ручная проверка игрового поведения, UI, прогресса, экономики и боев;
- тестеры-программисты: поиск причин багов в коде, проверка данных, консольная отладка и регрессия после правок.

## 1. Краткое описание проекта

Tanks Wars - браузерная 2D-игра про танковые бои. Проект сделан как статическое приложение без сборщика: `index.html` подключает CSS, данные из `data.csv` и JavaScript-файлы из папки `js`.

Основные зоны игры:

- ангар: выбранный танк, характеристики, нижняя лента танков, верхняя панель ресурсов;
- дерево наций: исследование и покупка техники;
- улучшения: быстрый экран исследования от выбранного танка;
- магазин: контейнеры, премиум-танки, золото за рекламу;
- профиль: имя игрока, ID, статистика, медали, список танков;
- настройки: графика, боевой UI, маркеры, полноэкранный режим;
- события: прогресс временного события;
- бой: canvas-карта, танки, снаряды, базы, мини-карта, награды.

## 2. Запуск и окружение

Рекомендуемый запуск - через локальный HTTP-сервер из корня проекта. Это важно, потому что `data.csv` загружается через `fetch("./data.csv")`; при открытии `index.html` через `file://` браузер может заблокировать загрузку CSV, и игра перейдет на ограниченный `fallbackTanks`.

Примеры запуска:

```powershell
python -m http.server 8000
```

или любым статическим сервером. После запуска открыть:

```text
http://localhost:8000/
```

Минимальная матрица браузеров:

- Chrome / Edge: основной браузер для регрессии;
- Firefox: проверка canvas, fullscreen, localStorage;
- мобильная эмуляция в DevTools: ширины 360, 390, 768 px;
- desktop: 1366x768, 1920x1080.

## 3. Структура проекта

- `index.html` - DOM-каркас, подключение скриптов.
- `css/styles.css` - вся верстка и визуальные стили.

- `js/02-hangar-tech-tree.js` - ангар, лента танков, дерево наций, исследование и покупка.

- `js/08-startup.js` - старт игры, старт/стоп боя, обработчики клавиатуры и мыши.

## 4. Хранилище и сброс состояния

Игра хранит прогресс локально:

- ресурсы: `playerResources`;
- профиль: `playerProfile`;
- статистика: `playerStats`;
- настройки: `gameSettings`;
- опыт танков: cookie вида `tank_<id>_exp`;
- состояние танков: cookie вида `tank_<id>_state`;
- событие: cookie `victory_day_2026_wins`, `victory_day_2026_claimed`.

Перед чистой регрессией нужно очистить site data в DevTools или вручную удалить localStorage/cookies для локального адреса.

Ожидаемое состояние после чистого старта: игрок получает начальный доступный танк, ресурсы равны значениям по умолчанию, настройки стандартные.

## 5. Данные `data.csv`

Каждая строка описывает один танк. Важные поля по индексам:

- `A` / `cells[0]`: название танка;
- `B-D` / `cells[1..3]`: типы снарядов;
- `E-G` / `cells[4..6]`: урон снарядов;
- `H` / `cells[7]`: уровень;
- `I` / `cells[8]`: нация;
- `J-L` / `cells[9..11]`: ссылки исследования;
- `M` / `cells[12]`: цена исследования в опыте;
- `N` / `cells[13]`: цена покупки в серебре;
- `O` / `cells[14]`: класс танка;
- `P` / `cells[15]`: HP;
- `Q` / `cells[16]`: перезарядка;
- `R` / `cells[17]`: задержка поворота корпуса;
- `S` / `cells[18]`: задержка движения;
- `T` / `cells[19]`: пробитие первого снаряда;
- `U` / `cells[20]`: пробитие второго снаряда;
- `V` / `cells[21]`: пробитие третьего снаряда;
- `W` / `cells[22]`: средняя броня;
- `X` / `cells[23]`: шанс пробития;
- `Y` / `cells[24]`: тип орудия;
- `Z` / `cells[25]`: снарядов за выстрел;
- `AA` / `cells[26]`: размер магазина;
- `AB` / `cells[27]`: разброс;
- `AC` / `cells[28]`: флаг доступности / вид танка;
- `AD` / `cells[29]`: дополнительные способности.

Флаг `AD`:

- `0`: будущий танк, отображается как недоступный и не должен выбираться ботами;
- `1`: обычный танк дерева;
- `2`: премиум/контейнерный танк;
- `3`: танк для режима разработчика, даётся при влючении режима;
- пусто: допускается как обычная строка для совместимости, но требует отдельной проверки.

Типы снарядов:

- `ББ`, `ПБ`, `КС`, `ОФ` - обычные типы;
- `ОГОНЬ` или `FIRE` - огнеметный поток;
- `ПТУР` или `ATGM` - управляемая ракета, управление активно первые 15 секунд полета.

## 6. Управление

В ангаре:

- выбор танка - клик по нижней ленте;
- экраны слева - клик по иконкам;
- режим боя - кнопки в верхней панели;
- старт боя - кнопка боя в верхней панели.

В бою:

- `W/A/S/D` или русская раскладка `Ц/Ф/Ы/В` - движение;
- мышь - наведение корпуса/башни;
- ЛКМ - выстрел;
- `1/2/3` - выбор снаряда;
- `4` - режим залпа/барабана для подходящих орудий;
- `Tab` - показать/скрыть списки команд;
- `H` - скрыть/показать обучение;
- `G` - артиллерийский обзор для САУ;
- стрелки влево/вправо - переключение наблюдаемого союзника после смерти.

## 7. Режимы боя

- Ротный бой: 7 на 7. Победа при уничтожении всех врагов или захвате базы.
- Взводный бой: 5 на 5. Те же условия, меньше состав.
- Дуэль: 1 на 1. Быстрая проверка стрельбы, урона, победы/поражения.
- Охота на командира: 7 на 7. Победа при уничтожении вражеского командира; смерть игрока-командира дает поражение.
- Война: 15 на 15. Нужно захватить 4 точки, затем базу врага. Танки возрождаются до завершения боя.

Режимы `commander` и `war` доступны только во время события 7-15 мая по локальной дате браузера.

## 8. Боевые механики для проверки

Танк:

- HP берется из `data.csv`;
- скорость зависит от задержки движения и класса;
- ПТ/ПТ-САУ/САУ не имеют вращающейся башни;
- колесные классы имеют отдельную модель руления.

Стрельба:

- снаряд создается у дула;
- обычные снаряды летят прямо;
- ОФ может наносить частичный урон без пробития;
- ББ/ПБ могут пробивать некоторые камни;
- огонь наносит потоковый урон при удержании ЛКМ;
- ПТУР доворачивает к курсору игрока или цели бота первые 15 секунд, затем летит прямо.

Перезарядка:

- обычное орудие блокирует выстрел до окончания КД;
- магазинные типы используют `clipAmmo`, `clipSize`, `clipReloadTimer`;
- многоснарядный залп отображает множитель урона;
- индикатор КД у курсора зависит от настройки `showReloadIndicator`.

Боты:

- выбирают танки того же уровня, кроме танков с `AD = 0`;
- ищут цель среди живых противников;
- учитывают класс: тяжелые давят направление, ПТ/САУ держат дистанцию, поддержка держится за тяжелыми;
- в режиме войны получают цели по точкам и базам.

Захват:

- в обычных режимах база захватывается при присутствии танков одной команды;
- спорный захват должен тормозиться или сбрасываться;
- несколько танков ускоряют захват;
- в режиме войны сначала захватываются точки, база становится финальной целью.

Награды:

- после боя начисляются опыт и серебро;
- статистика игрока и выбранного танка обновляется один раз за результат;
- победа дает больший множитель;
- медали зависят от условий боя.

## 9. Основные ручные чек-листы

### 9.1 Первый запуск

- Игра показывает загрузочный экран и затем ангар.
- `data.csv` загружен, список танков больше fallback-набора из 7 танков.
- В ангаре видны выбранный танк, статы, ресурсы, кнопки экранов.
- Ошибок в консоли нет.

### 9.2 Ангар и выбор танка

- Доступный танк выбирается кликом.
- Недоступный танк не становится выбранным для боя.
- Премиум/будущие танки имеют отличимое оформление.
- Изображение танка меняется вместе с выбором.
- Характеристики соответствуют данным CSV.

### 9.3 Дерево наций

- Открывается экран наций.
- Переключаются вкладки наций.
- Линии исследований не смещены после ресайза.
- Недостаток опыта/серебра не позволяет исследовать/купить.
- После исследования списывается опыт и меняется состояние танка.
- После покупки списывается серебро и танк становится доступным.

### 9.4 Улучшения

- Экран показывает выбранный танк и доступные цели исследования.
- Цены совпадают с CSV.
- Нет пустого экрана для танков без продолжения ветки.
- Возврат назад восстанавливает ангар.

### 9.5 Магазин

- Контейнер недоступен при золоте меньше цены.
- При покупке списывается 500 золота.
- Выдается 3 ресурсных приза или танк с шансом 5%.
- Дубликат танка компенсируется золотом.
- Премиум-танки покупаются за золото или чертежи.
- Кнопка рекламы корректно обрабатывает отсутствие SDK.

### 9.6 Профиль

- Имя сохраняется после перезагрузки.
- ID имеет стабильный формат.
- Общая статистика обновляется после боя.
- Статистика по танкам обновляется по выбранному танку.
- Медали открываются при выполнении условий.

### 9.7 Настройки

- Яркость, контраст, насыщенность меняют боевую карту.
- Масштаб UI влияет на HUD боя.
- Полоски HP, маркеры команд, индикатор КД скрываются/показываются.
- Полноэкранный режим включается и выключается.
- Сброс возвращает значения по умолчанию.
- Настройки сохраняются после перезагрузки.

### 9.8 Бой

- Все режимы стартуют с правильным количеством танков.
- Игрок появляется на союзном спавне.
- Враги появляются на противоположной стороне.
- Камера следует за игроком.
- Мини-карта показывает союзников, засвеченных врагов, базы/точки.
- Победа и поражение завершают бой один раз.
- Кнопка назад корректно возвращает в ангар.

### 9.9 Снаряды и урон

- Все 3 слота снарядов выбираются клавишами и кликом.
- Урон попадает в статистику только при попадании игрока.
- Уничтожение врага увеличивает счетчик фрагов.
- ОФ без пробития наносит частичный урон.
- Огонь наносит непрерывный урон при удержании.
- ПТУР управляется курсором первые 15 секунд и затем перестает доворачивать.

### 9.10 Режим войны

- На карте есть 4 точки и 2 базы.
- Точки захватываются за 10 секунд базово.
- После захвата всех точек команда может брать базу врага.
- Мертвые танки возрождаются примерно через 4 секунды.
- Бой завершается при захвате финальной базы.

## 10. Регрессия после изменения данных CSV

После любых изменений `data.csv` проверить:

- нет пустых названий танков;
- у каждого танка есть уровень, класс, HP, перезарядка;
- ссылки исследования ведут на существующие названия;
- изображения `img/<name>.png`, `img/korpus/<name>.png`, `img/bashnya/<name>.png` существуют для танков, которые должны отображаться;
- `AD = 0` не попадает ботам;
- `AD = 2` появляется в магазине/контейнерах;
- типы снарядов не содержат лишних пробелов и опечаток;
- десятичные значения с запятой корректно парсятся.

## 11. Регрессия после изменения боевой логики

Минимальный набор:

- Дуэль: выстрел, попадание, победа, награда.
- Ротный бой: боты двигаются, стреляют, база захватывается.
- Охота на командира: смерть вражеского командира дает победу.
- Война: захват точек, респавн, захват базы.
- САУ: `G` включает карту, враги видимы.
- ПТ/ПТ-САУ: корпус доворачивает к цели, башня не вращается.
- Огнеметный снаряд: удержание ЛКМ не ломает КД.
- ПТУР: управление 15 секунд, затем прямой полет.

## 13. Типовые признаки багов

Критические:

- игра не стартует;
- бой не завершается;
- прогресс или ресурсы теряются после перезагрузки;
- ошибка JavaScript останавливает игровой цикл;
- игрок получает бесконечные ресурсы без условия;
- танк с `AD = 0` появляется у бота.

Высокий приоритет:

- невозможно купить/исследовать при достаточных ресурсах;
- урон не засчитывается;
- победа засчитывается как поражение;
- неверный респавн в режиме войны;
- снаряды проходят через непредусмотренные препятствия;
- UI закрывает важные элементы боя.

Средний приоритет:

- неверные числа в UI;
- визуальные смещения линий дерева;
- мини-карта показывает неверные маркеры;
- настройки применяются только после перезагрузки;
- пропадает изображение танка.

Низкий приоритет:

- опечатки;
- мелкие несовпадения отступов;
- неидеальные анимации;
- предупреждения консоли без влияния на сценарий.

## 15. Чек-лист перед релизной проверкой

- Запустить с чистым хранилищем.

- Пройти по всем боковым экранам.

- Проверить начисление опыта, серебра, статистики.
- Проверить магазин и профиль через developer mode.
